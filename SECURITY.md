# Security Policy

## 対応方針

脆弱性や認証・権限制御の問題を見つけた場合は、公開 Issue ではなく非公開で連絡してください。

## 連絡方法

- GitHub の private な連絡手段が使える場合はそちらを優先
- それが難しい場合は、公開 Issue に詳細を書かず「連絡先を知りたい」とだけ記載

## 報告時にあると助かる情報

- 影響範囲
- 再現手順
- 想定される悪用方法
- 可能なら暫定回避策

## 注意

- API キー、トークン、個人情報は公開 Issue や PR に貼り付けないでください
- Firebase Security Rules や Cloud Functions の認可不備は重要度高めで扱います
